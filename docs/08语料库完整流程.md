## 模块说明

### 1. 数据源采集 (data-collection/)

#### 1.1 GitHub代码采集
- **文件**: `code-to-corpus/`
- **功能**: 从GitHub爬取代码仓库，提取代码文本
- **输出格式**:
  ```json
  {
    "来源": "github",
    "仓库名": "author/repo",
    "文件名": "main.py",
    "ext": ".py",
    "path": "repo/src/main.py",
    "size": 1234,
    "原始编码": "utf-8",
    "md5": "abc123...",
    "text": "代码内容...",
    "时间": "20250318"
  }
  ```

#### 1.2 PDF文档采集
- **文件**: `pdf-to-corpus/`
- **功能**: 处理PDF文档，提取文本内容
- **输出格式**:
  ```json
  {
    "section": "章节标题",
    "content": "章节内容..."
  }
  ```

### 2. 文本提取 (text-extraction/)

#### 2.1 PDF提取
- **工具**: MinerU, PyMuPDF, pdfplumber
- **支持**: OCR识别、表格提取、图片处理
- **输出**: Markdown格式

#### 2.2 代码提取
- **工具**: charset_mnbvc (编码检测)
- **支持**: 多种编程语言、多种编码格式

### 3. 数据清洗 (data-cleaning/)

#### 3.1 文本清洗
- 移除图片、链接、目录、乱码
- 移除参考文献、页眉页脚
- 修复编码问题

#### 3.2 语义去重
- 使用Sentence-BERT进行语义相似度计算
- 可配置相似度阈值（默认0.9）

### 4. 格式转换 (format-conversion/)

#### 4.1 TRL训练格式（新增）

**SFT格式 - Conversational**:
```json
{
  "messages": [
    {"role": "user", "content": "用户问题"},
    {"role": "assistant", "content": "助手回答"}
  ]
}
```

**SFT格式 - Text**:
```json
{
  "text": "完整的文本内容..."
}
```

**SFT格式 - Prompt-Completion**:
```json
{
  "prompt": "问题描述或上下文",
  "completion": "期望的输出"
}
```

**DPO格式**:
```json
{
  "prompt": "用户问题",
  "chosen": "优选回答",
  "rejected": "拒绝回答"
}
```

### 5. 数据验证 (data-validation/)

#### 5.1 格式验证
- JSON格式检查
- 必需字段检查
- 数据类型验证

#### 5.2 质量评估
- 文本长度统计
- 语言分布
- 质量分数

### 6. 数据集划分 (dataset-split/)

```python
from datasets import load_dataset

# 加载并划分数据集
dataset = load_dataset("json", data_files="corpus.jsonl")
dataset = dataset["train"].train_test_split(test_size=0.1, seed=42)
# 输出: {"train": ..., "test": ...}
```

## 完整流程示例

### 示例1: 代码语料处理

```bash
# 1. 准备仓库列表
echo "12345,https://github.com/user/repo.git" > repos_list.txt

# 2. 爬取元数据
python code-to-corpus/auto-metedata.py -p github --start 1 --end 1000000

# 3. 下载仓库
python code-to-corpus/github_downloader.py

# 4. 转换为SFT格式
python corpus-tools/convert_to_sft_format.py \
    --input output/jsonl/githubcode.0.jsonl \
    --output output/sft/code_sft.jsonl \
    --format text

# 5. 验证数据
python corpus-tools/validate_dataset.py \
    --input output/sft/code_sft.jsonl \
    --format sft
```

### 示例2: PDF文档处理

```bash
# 1. 提取PDF内容
python pdf-to-corpus/ai4e_refinetext/pdf_extractor.py

# 2. 清洗Markdown
python pdf-to-corpus/ai4e_refinetext/markdown_cleaner.py \
    --input output/markdown/ \
    --output output/cleaned/

# 3. 语义去重
python pdf-to-corpus/ai4e_refinetext/semantic_deduplicator.py \
    --input output/cleaned/merged.txt \
    --output output/dedup/dedup.txt

# 4. 转换为SFT格式
python corpus-tools/convert_to_sft_format.py \
    --input output/dedup/dedup.txt \
    --output output/sft/pdf_sft.jsonl \
    --format text

# 5. 划分数据集
python corpus-tools/split_dataset.py \
    --input output/sft/pdf_sft.jsonl \
    --output output/final/ \
    --test-size 0.1
```

## 与HuggingFace训练的集成

### 使用TRL进行训练

```python
from datasets import load_dataset
from trl import SFTTrainer, SFTConfig
from peft import LoraConfig

# 加载处理好的数据集
dataset = load_dataset("json", data_files="output/sft/train.jsonl")

# 创建训练器
trainer = SFTTrainer(
    model="Qwen/Qwen2.5-7B",
    train_dataset=dataset["train"],
    args=SFTConfig(
        output_dir="output/model",
        num_train_epochs=3,
        per_device_train_batch_size=4,
        learning_rate=2e-5,
        push_to_hub=True,
        hub_model_id="username/your-model",
    ),
    peft_config=LoraConfig(r=16, lora_alpha=32),
)

# 开始训练
trainer.train()
```

## 常见问题

### Q1: 如何处理多语言数据？
建议按语言分开处理，分别训练或使用多语言基座模型。

### Q2: 数据量需要多大？
- 微调：通常需要1K-10K高质量样本
- 预训练：需要更大规模数据（百万级以上）

### Q3: 如何评估数据质量？
使用提供的验证工具，检查格式、长度分布、语言分布等指标。

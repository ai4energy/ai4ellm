# AI4ELLM 容器化部署指南

## 一、架构差异对比

| 项目 | NVIDIA (5090) | 海光 DCU |
|------|---------------|----------|
| 基础镜像 | `nvidia/cuda:12.1-...` | `rocm/dev-ubuntu-22.04:6.0` |
| GPU 驱动 | CUDA 12.x | ROCm 6.0 (HIP) |
| 设备驱动 | `nvidia` | `amd` |
| GPU 调用 | `--gpus all` | `--device=/dev/dri` |
| Python 版本 | 3.12 | 3.12（一致） |

**好消息：我们的 ai4ellm 项目主要是语料处理和 API 调用，不直接跑 GPU 推理。所以 CUDA/ROCm 差异对语料流水线影响极小。**

只有后续做**模型微调训练**时才需要 GPU，那时 Dockerfile 才需要区分 ROCm/CUDA 的 PyTorch 版本。

## 二、构建镜像

### 海光 DCU (ROCm)
```bash
docker build -f Dockerfile.rocm -t ai4ellm:rocm .
```

### NVIDIA GPU (CUDA)
```bash
docker build -f Dockerfile.cuda -t ai4ellm:cuda .
```

## 三、本地运行

### 海光 DCU
```bash
docker run --rm \
  --device=/dev/dri \
  -v ./data:/app/data \
  -v ./output:/app/output \
  -e DEEPSEEK_API_KEY=sk-xxx \
  -e DASHSCOPE_API_KEY=sk-xxx \
  ai4ellm:rocm
```

### NVIDIA GPU
```bash
docker run --rm \
  --gpus all \
  -v ./data:/app/data \
  -v ./output:/app/output \
  -e DEEPSEEK_API_KEY=sk-xxx \
  -e DASHSCOPE_API_KEY=sk-xxx \
  ai4ellm:cuda
```

## 四、Docker Compose

```bash
# 海光
docker compose --profile rocm up

# NVIDIA
docker compose --profile cuda up
```

## 五、K8s 部署（曙光）

### K8s Secret（API 密钥）
```bash
kubectl create secret generic ai4ellm-api-keys \
  --from-literal=deepseek-api-key=sk-xxx \
  --from-literal=dashscope-api-key=sk-xxx
```

### 挂载存储
```yaml
volumes:
  - name: ai4ellm-data
    persistentVolumeClaim:
      claimName: ai4ellm-data-pvc
  - name: ai4ellm-secrets
    secret:
      secretName: ai4ellm-api-keys
```

### GPU 资源申请
```yaml
resources:
  limits:
    # 海光 DCU
    hygon.com/dcu: 1
    # 或 NVIDIA
    # nvidia.com/gpu: 1
```

## 六、启动脚本（挂载进容器）

K8s 容器启动脚本示例：

```bash
#!/bin/bash
# start_pipeline.sh

# 设置 API 密钥
export DEEPSEEK_API_KEY=${DEEPSEEK_API_KEY}
export DASHSCOPE_API_KEY=${DASHSCOPE_API_KEY}

# 运行语料处理流水线
echo "📚 运行预训练语料处理..."
python scripts/run_pretrain_pipeline.py --config configs/pretrain.yaml

echo "📝 运行 SFT 语料生成..."
python scripts/run_sft_pipeline.py --config configs/sft.yaml

echo "✅ 全部完成"
```

## 七、关键注意事项

1. **语料处理不依赖 GPU** — 主要是文本提取、清洗、去重、API 调用，CPU 就能跑
2. **模型微调才需要 GPU** — 后续训练阶段需要挂载 GPU 节点
3. **API 密钥通过 Secret 注入** — 不要写死在镜像里
4. **数据持久化通过 PVC** — 语料和输出通过存储卷持久化
5. **海光 DCU 的 ROCm 兼容 PyTorch** — 训练阶段用 `pytorch-rocm` 版本
